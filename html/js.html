Область видимости.

Это часть программы, в которой мы можем обратиться к переменной, функции или объекту.

Глобальная область видимости (Global Scope).

Все переменные и константы, которые объявлены вне функций, являются глобальными. Они доступны из любого места программы.

Пример Global Scope

var a = 5;
let b = 8;
const c = 9;
function sum() {
return a + b + c;
}
console.log(sum());

Локальная область видимости (Local Scope).

Переменные, объявленные внутри функций, становятся локальными и рассматриваются в своей локальной области. Локальная область видимости может быть разделена на область видимости функции и область видимости блока.

Пример Local Scope

function sum() {
var a = 5;
let b = 8;
const c = 9;
return a + b + c;
}
console.log(sum());

Область видимости функции (Function Scope).

Когда мы объявляем переменную в функции, переменная видна только внутри функции. Мы не можем получить к ней доступ вне функции.

Пример Function Scope


function sum3() {
var a = 5;
let b = 8;
const c = 9;
}
console.log(a);
console.log(b);
console.log(c);

Область видимости блока (Block Scope).

Это область в условиях (if и switch) или циклов (for, while). const и let позволяют разработчикам объявлять переменные в области видимости блока, что означает, что эти переменные существуют только в соответствующем блоке. При этом var буде виден ещё и в области функции.

Пример Block Scope

function sum() {
if (true) {
var a = 5;
let b = 8;
const c = 9;
console.log(a);
console.log(b);
console.log(c);
}
console.log(a);
}
sum();

Лексическая область видимости (Lexical Scope).

Ещё один момент, о котором стоит упомянуть - это лексическая область. Лексическая область означает, что дочерняя область имеет доступ к переменным, определенным в родительской области. Дочерние функции лексически связаны с контекстом исполнения их родителей.

Пример Lexical Scope

function sum() {
var a = 5;
let b = 8;
const c = 9;
function sum2() {
console.log(a);
console.log(b);
console.log(c);
}
sum2();
}
sum();

Динамическая область видимости (Dynamic Scope).

Не связана с тем, как и где объявляются функции и области, а связана с тем, откуда они вызываются. Другими словами, цепочка областей видимости основана на стеке вызовов, а не на вложении областей видимости в коде.

Пример Dynamic Scope

function sum() {
var a = 5;
let b = 8;
const c = 9;
function sum2() {
console.log(a);
console.log(b);
console.log(c);
}
sum2();
}
sum();


Переменные

Для хранения данных в программе используются переменные. Переменные предназначены для хранения каких-нибудь временных данных или таких данных, которые в процессе работы могут менять свое значение.

var

Область видимости переменных var ограничивается либо функцией, либо, если переменная глобальная, то скриптом. Такие переменные доступны за пределами блока.
Объявления переменных var обрабатываются в начале выполнения функции (или запуска скрипта, если переменная является глобальной). Другими словами, переменные var считаются объявленными с самого начала исполнения функции вне зависимости от того, в каком месте функции реально находятся их объявления (при условии, что они не находятся во вложенной функции).

let

Локальная переменная с областью видимости, ограниченной текущим блоком кода. Является перезаписіваемой и её можно обновлять, но не переопределять.

const

Значение констант не может быть изменено новым присваиванием, а также не может быть переопределено. Константы подчиняются области видимости уровня блока.

Операторы сравнения.

==

Оператор равенства сравнивает два значения, и если они равны, возвращает true, иначе возвращает false: x == 5

===

Оператор тождественности также сравнивает два значения и их тип, и если они равны, возвращает true, иначе возвращает false: x === 5

!=

Сравнивает два значения, и если они не равны, возвращает true, иначе возвращает false: x != 5

!==

Сравнивает два значения и их типы, и если они не равны, возвращает true, иначе возвращает false: x !== 5

>

Сравнивает два значения, и если первое больше второго, то возвращает true, иначе возвращает false: x > 5

<

Сравнивает два значения, и если первое меньше второго, то возвращает true, иначе возвращает false: x < 5

>=

Сравнивает два значения, и если первое больше или равно второму, то возвращает true, иначе возвращает false: x >= 5

<=

Сравнивает два значения, и если первое меньше или равно второму, то возвращает true, иначе возвращает false: x <= 5

Логические операции.

let x = 50;

console.log(result); //true

!

Возвращает true, если операция сравнения возвращает false.
let x = 50;
let result = !(x > 20);

Циклы.

Циклы позволяют в зависимости от определенных условий выполнять некоторое действие множество раз.

for

Цикл for состоит из трех частей:
1. Объявления цикла - let i = 0 - создает и инициализирует счетчик.
2. Условие, при котором будет выполняться цикл: i < 5.
3. Сам счетчик.

Пример for i++

i++ или i+= 1

for (let i = 0; i < 5; i++) {
console.log(i);
}

Пример for i--

i-- или i-= 1

for (let i = 5; i > 0; i--) {
console.log(i);
}

Пример for четные числа

for (let i = 0; i < 10; i += 2) {
console.log(i);
}

Пример for нечетные числа

for (let i = 1; i < 10; i += 2) {
console.log(i);
}

Пример for вложенные циклы

for (let i = 1; i <= 2; i++) {
for (let j = 1; j <= 2; j++) {
console.log(i * j);
}
}

while

Цикл while выполняется до тех пор, пока некоторое условие истинно.

Пример while

let i = 1;
while (i <= 4) {
console.log(i);
i++;
}

do while

В цикле do сначала выполняется код цикла, а потом происходит проверка условия в инструкции while. И пока это условие истинно, цикл повторяется.

Пример do while

let i = 1;
do {
console.log(i);
i++;
} while (i <= 4);

Операторы continue

Позволяет продолжить цикл до определённого момента.

Пример continue

Когда значение переменной i станет равным 6, то в цикле конструкция if(i===6) continue завершит текущую итерацию, далее идущие инструкции цикла не будут выполняться, и произойдет переход к следующей итерации.

for(let i=1; i <= 5; i++){
if(i===6) continue;
console.log(i);
}

Операторы break

Позволяет прервать цикл.

Пример break

Данный цикл увеличивает переменную i c 1 до 10. Однако поскольку в блоке цикла происходит поверка if(i===6) break;, то, когда значение переменной i достигнет 6, то данное условие прервет выполнение цикла с помощью оператора break.

for(let i=1; i <= 10; i++){
if(i===6) break;
console.log(i);
}

for in

Предназначен главным образом для перебора объектов. Этот цикл перебирает все свойства объекта.

Пример for in

const people = { name: 'Vasya', age: 22, budget: 4500 };
for (let prop in people) {
console.log(prop);
}

for of

Предназначен для перебора наборов данных. Например массивов.

Пример for of

const people = [
{ name: 'Vasya', age: 22, budget: 4500 },
{ name: 'Petya', age: 21, budget: 4200 },
];
for (let person of people) {
console.log(person.name);
console.log(person.age);

}

Любые текстовые данные в JavaScript считаются строками. Это примитивный тип, но язык позволяет работать с ним так, будто он является объектом. В том числе — использовать встроенные в JS методы строк.

this

Значение this определяется тем, каким образом вызвана функция. В JavaScript ключевое слово this относится к объекту, к которому оно принадлежит.
Оно имеет разные значения в зависимости от того, где используется:
- в методе, this относится к объекту, по отношению к которому вызван метод;
- в глобальном контексте выполнения this ссылается к глобальному объекту;
- в функции, значение this зависит от того, каким образом вызвана функция;
- в обычной функции this относится к глобальному объекту;
- в стрелочных функциях, this привязан к окружению, в котором была создана функция;
- в строгом режиме функции, this не определен, если предварительно он не был установлен в контексте исполнения;
- в событии this относится к элементу, который получил событие;
- такие методы, как call() и apply() могут ссылать this на любой объект.

Вставляет символ доллара $.

Флаги.

i

С этим флагом поиск не зависит от регистра: нет разницы между A и a

g

С этим флагом поиск ищет все совпадения, без него – только первое.

m

Многострочный режим.

s

Включает режим «dotall», при котором точка . может соответствовать символу перевода строки \n

u

Включает полную поддержку Юникода. Флаг разрешает корректную обработку суррогатных пар.

y

Режим поиска на конкретной позиции в тексте.

Символьные классы.

\d

Цифра: символ от 0 до 9. Аналог [0-9].
str.match(/\d/g, '')

\D

Не цифры. Аналог [^0-9]

\s

Пробельные символы, табы, новые строки.
str.match(/\s/gi, '')

\S

Все, кроме \s.

\w

Латиница, цифры, подчёркивание '_'.
str.match(/\w/gi, '')

\W

Все, кроме \w.

.

Любой символ, если с флагом регулярного выражения s, в противном случае любой символ, кроме перевода строки \n.

\n

Соответствует символу перевода строки (U+000A).

\r

Соответствует символу возврата каретки (U+000D).

^

Совпадение с началом текста.
let str = "We will, we will rock you";
console.log(/^We/.test(str) )
// true

$

Совпадение с концом текста.
let str = "We will, we will rock you";
console.log(/you$/.test(str))
// true

*

Соответствует предыдущему символу повторенному 0 или более раз.

+

Соответствует предыдущему символу повторенному 1 или более раз.

?

Соответствует предыдущему символу повторенному 0 или 1 раз.
let str = '123abc';
console.log(str.match(/\d+/)); // 123
console.log(str.match(/\d+?/)); // 1

(x)

Соответствует 'x' и запоминает это соответствие. Это называется захватывающие скобки.
let str = 'We will, we will rock you';
console.log(str.match(/(wi)/));
// (2) 'wi', 'wi'

(?:x)

Соответствует 'x' но не запоминает соответствие. Это называется не-захватывающие скобки.
let str = 'We will, we will rock you';
console.log(str.match(/(?:wi)/));
// wi

x(?=y)

Соответствует 'x' только если за 'x' следует 'y'. Это называется упреждение.
let str = 'We will, we will rock you';
console.log(str.match(/we(?: will)/));
// we will

x(?!y)

Соответствует 'x' только если за 'x' не следует 'y'. Это называется отрицательное упреждение.

x|y

Соответствует либо 'x' либо 'y'.
let str = 'We will, we will rock you';
console.log(str.match(/We|will/));
// We

{n}

n - положительное целое. Соответствует ровно n вхождениям предыдущего символа.
let str = 'We will, we will rock you';
console.log(str.match(/l{2}/));
// ll

{n,m}

m и n - положительные целые. Соответствует как минимум n и максимум m вхождениям предыдущего символа. При m=n=1 пропускается.
let str = 'greeeeen';
console.log(str.match(/e{1,3}/));
// eee

[xyz]

Набор символов. Соответствует любому символу из перечисленных. Можно указать диапазон символов, используя тире.
let str = 'We will, we will rock you';
console.log(str.match(/[abcd]/));
// c

[^xyz]

Инвертированный или дополняющий набор символов. Это означает соответствие всему, что не в скобках. Можно указать диапазон символов с помощью тире.
let str = 'We will, we will rock you';
console.log(str.match(/[^abcd]/));
// W

[\b]

Соответствует бэкспейсу (U+0008).

\b

Соответствует границе слова. Граница слова соответствует позиции, где за символом слова не следует другой символ слова или предшествует ему.
let str = 'We will, we will rock you';
console.log(str.match(/\b/));
// ' '

\B

Соответствует несловообразующей границе. Несловообразующая граница соответствует позиции, в которой предыдущий и следующий символы являются символами одного типа: либо оба должны быть словообразующими символами, либо несловообразующими.
let str = 'We will, we will rock you';
console.log(str.match(/\B..../));
// e wi

\f

Соответствует символу прогона страницы (U+000C). Особый символ управления печатью.

\t

Соответствует символу горизонтальной табуляции (U+0009).

\v

Соответствует символу вертикальной табуляции (U+000B).

\0

Соответствует символу NULL (U+0000).

\xhh

Соответствует символам кода hh (две шестнадцатеричные цифры).

\uhhhh

Соответствует символам кода hhhh (четыре шестнадцатеричные цифры).

Symbol

Это уникальный и неизменяемый тип данных, который может быть использован как идентификатор для свойств объектов. Синтаксис выглядит так: let symbol = Symbol();
Каждый символ является уникальным и никогда не будет равен другому, даже если описание одинаковое.

Пример Symbol

let symbolOne = Symbol('One');
let symbolTwo = Symbol('One');
console.log(symbolOne === symbolTwo);

Методы.

Symbol.for(key)

Метод Symbol.for() сначала ищет символ с One ключом в глобальном реестре символов. Он возвращает существующий символ, если он есть. В противном случае Symbol.for() метод создает новый символ, регистрирует его в глобальном реестре символов с указанным ключом и возвращает символ.

Пример Symbol.for

let symbolOne = Symbol.for('One');
let symbolTwo = Symbol.for('One');
console.log(symbolOne === symbolTwo);

Symbol.keyFor(sym)

Позволяет получить ключ. Если ключ отсутствует тогда выведет undefined.

Пример Symbol.keyFor

let symbolOne = Symbol.for('One');
let symbolTwo = Symbol('One');
console.log(Symbol.keyFor(symbolOne));
console.log(Symbol.keyFor(symbolTwo));

Итераторы.

Это объект, предназначенный для перебора другого объекта. Практически везде, где нужен перебор, он осуществляется через итераторы. Это включает в себя не только строки, массивы, но и вызов функции с оператором spread f(...args), и многое другое.

next()

Метод который при каждом вызове возвращает очередное значение и проверяет, окончен ли перебор. Также у него присутствует два свойства:
value – очередное значение.
done – равно false если есть ещё значения, и true – в конце.

Пример итератор

let range = {
from: 1,
to: 5,
};
range[Symbol.iterator] = function () {
let current = this.from;
let last = this.to;
return {
next() {
if (current <= last) {
return { done: false, value: current++ };
} else {
return { done: true };
}
},
};
};
for (let num of range) {
console.log(num);
}

yield 2;
return 3;
}
* - означает, это генератор, а не обычная функция.
yield - оператор возвращает значение и приостанавливает выполнение функции.
Далее следует поместить генератор в переменную.
let generator = generateSequence();

Пример генератор

yield 3;
console.log('finish');
}

console.log(generator.next());
console.log(generator.next());
console.log(generator.next());
console.log(generator.next());

Для того чтобы вывести в консоль данные из массива по очередно необходимо поставить *. Иначе мы выведем {value: Array(3), done: false}

Объекты в JavaScript можно организовать в цепочки так, чтобы свойство, не найденное в одном объекте, автоматически искалось бы в другом.

__proto__

Если у объекта присутствует ссылка __proto__ ведущая на другой объект, то когда будет вызвано какое-то свойство у данного объекта и оно не будет найдено, тогда поиск перейдёт по цепочке к связанному объекту через __proto__ .
На данный момент __proto__ считается устаревшим, и по стандарту оно должно поддерживаться только браузерами.

Пример proto

Рассмотрим пример: есть два объекта people и masha. Когда мы будем вызвать masha.eats - то мы его не найдём и перейдём по цепочке в people.

const people = {
eats: 'vegetarian',
};
const masha = {
sport: 'tennis',
__proto__: people,
};
console.log(masha.sport);
console.log(masha.eats);

второй вариант записи masha
const masha = {
sport: 'tennis',
};
masha.__proto__ = people;

Пример objectCreate

Асинхронность

Основной инструмент, который обрабатывает запросы параллельно с загрузкой веб-страницы. Сейчас невозможно представить интернет, где все запросы на сервер отправлялись бы с перезагрузкой страницы. Promise

Представляет собой обработчик асинхронных операций который возвращаемый объект с прекреплённым колбеком. Колбек может быть двух видов:
resolve - когда операция прошла успешно.
reject - если в процессе операции произошла ошибка.
Promise может находиться в трёх состояниях, каждое из которых имеет свой результат:
pending - Состояние ожидания. Которое в зависимости от колбека перетекает в другое состаяние. В данном случае результат undefined.
fulfilled - Состояние в которое переходит при колбеке resolve. В данном случае результат будет положительным и мы получим значение (value).
rejected - Состояние в которое переходит при колбеке reject. Ну и на конец результат будет отрицательный и мы получим (error).

Методы промисов.

Promise.all

Ожидает выполнения всех промисов и возвращает массив с результатами. Если любой из указанных промисов вернёт ошибку, то результатом работы Promise.all будет эта ошибка, результаты остальных промисов будут игнорироваться.

Пример Promise.all

Promise возвращает 10, 20 и 30 через 1, 2 и 3 секунды. Мы используем setTimeout() для имитации асинхронных операций:

}, 1 * 1000);
});
const p2 = new Promise((resolve, reject) => {
setTimeout(() => {
console.log('The second promise has resolved');
resolve(20);
}, 2 * 1000);
});
const p3 = new Promise((resolve, reject) => {
setTimeout(() => {
console.log('The third promise has resolved');
resolve(30);
}, 3 * 1000);
});
Promise.all([p1, p2, p3]).then((results) => {
const total = results.reduce((p, c) => p + c);
console.log(`Results: ${results}`);
console.log(`Total: ${total}`);
});

getUser();

Promise

const delay = (ms) => {
return new Promise((res) => setTimeout(() => res(), ms));
};
const user = 'https://jsonplaceholder.typicode.com/users';

return new Promise((res) => setTimeout(() => res(), ms));

Proxy.

Прокси это своеобразная обёртка в которую помещаются другие объекты. При этом можно производить различные манипуляции (добавлять, изменять, удалять) с свойствами.
let proxy = new Proxy(target, handler);
target — объект (массив, функция и т.д.) для которого создаётся прокси.
handler - тут описываются ловушки которые необходимы для данного прокси.

Функции обработчика.

.apply()

apply(target, thisArg, argArray)
thisArg – это контекст this.
argArray – список аргументов.

Пример Proxy apply

const info = (text) => `Name: ${text}`;
const infoProxy = new Proxy(info, {
apply(target, thisArg, argArray) {
return target.apply(thisArg, argArray);
},
});
console.log(infoProxy('Vasya'));

.construct()

construct(target, argumentsList, newTarget)
newTarget - конструктор, который был первоначально вызван.

Пример Proxy construct

class Person {
constructor(name, age) {
this.name = name;
this.age = age;
}
}
const PersonProxy = new Proxy(Person, {
construct(target, argArray, newTarget) {
return new target(...argArray);
},
});
const person = new PersonProxy('Vasya', 33);
console.log(person);

.defineProperty()

defineProperty(target, property, descriptor)
descriptor - дескриптор определяемого или изменяемого свойства.

Пример Proxy defineProperty

const person = {
name: 'Vasya',
age: 22,
};
const personProxy = new Proxy(person, {
deleteProperty(target, property, descriptor) {
return Reflect.defineProperty(target, property, descriptor);
},
});
Object.defineProperty(personProxy, 'work', {
value: 'programmer',
type: 'work',
});
console.log(personProxy);

const person = {

.get()

Является ловушкой для получения значения свойства.
get(target, property, receiver)
target – это оригинальный объект, который передавался первым аргументом в конструктор new Proxy.
property – имя свойства.
receiver – это сам объект прокси (или наследующий от него объект).

Пример Proxy get

const person = {
name: 'Vasya',
age: 22,
work: 'programmer',
};
const personProxy = new Proxy(person, {
get(target, property) {
return target[property];
},
});

.getOwnPropertyDescriptor()

Является ловушкой и позволяет просмотреть точное описание свойства состоящего из строкового имени и дескриптора свойства.

Пример Proxy getOwnPropertyDescriptor

const person = {
name: 'Vasya',
age: 22,
work: 'programmer',
};
const personProxy = new Proxy(person, {
getOwnPropertyDescriptor(target, prop) {
return {
enumerable: true,
configurable: true,

},
});
console.log(Object.keys(personProxy));

.set()

Данная ловушка срабатывает когда необходимо написать какое то свойство.

target – это оригинальный объект, который передавался первым аргументом в конструктор new Proxy.
property – имя свойства.
value – значение свойства.
receiver – это сам объект прокси (или наследующий от него объект).

const person = {
name: 'Vasya',
age: 22,
work: 'programmer',
};
const personProxy = new Proxy(person, {
set(target, property, value) {
target[property] = value;
return true;
},
});
personProxy.age = 30;
personProxy.work = 'freelancer';
console.log(personProxy);
