Работа с DOM.

Document Object Model.

DOM – это объектная модель документа, которую браузер создаёт в памяти компьютера на основании HTML-кода, полученного им от сервера. Если сказать по-простому, то HTML-код – это текст страницы, а DOM – это набор связанных объектов, созданных браузером при парсинге её текста.

Узел html образован элементом html. Узлы head и body имеют родительскую связь с html. Узел head связан с link, meta и title и является для них родителем. В свою очередь body связан с header, section и footer и является для них родителем. Элемент section имеет трёх детей div, p и a.
Не зная как устроено DOM-дерево и связи между узлами найти какой-то определенный элемент в нём будет достаточно затруднительно.

Навигация по DOM-элементам.

DOM позволяет нам делать что угодно с элементами и их содержимым, но для начала нужно получить соответствующий DOM-объект.
Все операции с DOM начинаются с объекта document. Это главная «точка входа» в DOM. Из него мы можем получить доступ к любому узлу.
Так выглядят основные ссылки, по которым можно переходить между узлами DOM.

document.documentElement

Самый верхний узел документа он соответствует тегу < html >.

document.head

Применяется к тегу < head >

document.body

Другой часто используемый DOM-узел – узел тега < body >. Для сокращения кода в дальнейшем используем переменную.
const bodyElement = document.body;

.childNodes

Содержит список всех детей, включая текстовые узлы. bodyElement.childNodes

.hasChildNodes

Для проверки есть ли у элемента дочерние узлы. bodyElement.hasChildNodes

.firstChild

Обеспечивают быстрый доступ к первому дочернему элементу. bodyElement.firstChild

.lastChild

Обеспечивают быстрый доступ к последнему дочернему элементу. bodyElement.lastChild

.previousSibling

Предыдущий узел того же родителя. bodyElement.previousSibling

.nextSibling

Следующий узел того же родителя. bodyElement.nextSibling

.parentNode

Родитель. bodyElement.parentNode


Навигационные свойства, описанные выше, относятся ко всем узлам в документе. В частности, в childNodes находятся и текстовые узлы и узлы-элементы и узлы-комментарии, если они есть.
Но для большинства задач текстовые узлы и узлы-комментарии нам не нужны. Мы хотим манипулировать узлами-элементами, которые представляют собой теги и формируют структуру страницы.
Поэтому давайте рассмотрим дополнительный набор ссылок, которые учитывают только узлы-элементы.

.children

Коллекция детей, которые являются элементами. bodyElement.children

.firstElementChild

Первый дочерний элемент. bodyElement.firstElementChild

.lastElementChild

Последний дочерний элемент. bodyElement.lastElementChild

.previousElementSibling

Предыдущий соседний элемент. bodyElement.previousElementSibling

.nextElementSibling

Следующий соседний элемент. bodyElement.nextElementSibling

.parentElement

Родительский элемент. bodyElement.parentElement

Поиск элементов.

.getElementById(id)

Находит элемент по идентификатору элемента, является статическим. document.getElementById(id)

.getElementsByName(name)

Находит элементы по имени, является динамичным (живым). document.getElementsByName(name)

.getElementsByTagName(name)

Находит элементы по имени тегов. Способен искать внутри элементов и является динамичным (живым). document.getElementsByTagName(name)

.getElementsByClassName(name)

Находит элементы под названием класса. Способен искать внутри элементов и является динамичным (живым). document.getElementsByClassName (name)

.querySelectorAll(selector)

Находит элементы соответствующему css селектору. Способен искать внутри элементов и является статическим. document.querySelectorAll(selector)
Поиск по тегу первого уровня вложенности. document.querySelectorAll(selector > selector)
Поиск по нескольким класам. document.querySelectorAll(selector, selector)
Поиск по вложенным класам. document.querySelectorAll(selector selector)

.querySelector(selector)

Находит первый элемент соответствующему css селектору. Способен искать внутри элементов и является статическим. document.querySelector(selector)

.closest

Ищет ближайшего предка который соответствует css селектору. document.querySelector(selector) .closest(selector)

.matches

Ничего не ищет, а проверяет, удавлетворяет ли элемент css селекторуб возвращает true или false.
const element = document.querySelectorAll('.first__div');
for (let el of element) {
if (el.matches('[class$="first__div_red"]')) {
console.log('Red');
} else if (el.matches('[class$="first__div_blue"]')) {
console.log('Blue');
}
}

Изменение свойств элементов HTML.

.innerHTML

Изменяет внутренний HTML элемент.
element.innerHTML = новое содержимое html

.outerHTML

Отличие от innerHTML состоит в том что кроме содержимого мы получаем объект целиком.
element.outerHTML = новое содержимое html

.insertAdjacentHTML

Позволяет вставлять произвольный HTML в любое место документа, в том числе и между узлами!
html - Строка HTML, которую нужно вставить.
where - Куда по отношению к elem вставлять строку. Всего четыре варианта:
beforeBegin – перед elem.
afterBegin – внутрь elem, в самое начало.
beforeEnd – внутрь elem, в конец.
afterEnd – после elem.
element.insertAdjacentHTML(where, html);

.insertAdjacentElement

Метод добавляет переданный элемент в DOM-дерево относительно элемента, вызвавшего метод.
element - Элемент, добавляемый в DOM-дерево.
where - Куда по отношению к elem вставлять строку. Всего четыре варианта:
beforeBegin – перед elem.
afterBegin – внутрь elem, в самое начало.
beforeEnd – внутрь elem, в конец.
afterEnd – после elem.
element.insertAdjacentElement(where, element);

.insertAdjacentText

Метод помещает заданный текстовый узел в указанную позицию относительно элемента, который передан в вызове метода.
element - Текст, который будет помещён в заданную позицию.
where - Куда по отношению к elem вставлять строку. Всего четыре варианта:
beforeBegin – перед elem.
afterBegin – внутрь elem, в самое начало.
beforeEnd – внутрь elem, в конец.
afterEnd – после elem.
element.insertAdjacentText(where, element);

.textContent

Позволяет получить непосредственно содержимое. При этом внесеные изменения передаются в виде строки полностью.
element.textContent = новое содержимое html

.style

Изменение стиля элемента HTML.
element.style.property = new style

Пример style

Начальный текст.

function changeStyle() {
const element = document.getElementById('style-text');
element.style.color = 'lightblue';
element.style.fontSize = '24px';
element.innerHTML = 'Конечный текст';
}

getComputedStyle

Позволяет получить все стили элемента.
getComputedStyle(element)

Пример getComputedStyle

Text
.div { font-size: 20px; color: white; margin: 10px 0;}

function getStyle() {
const element1 = document.querySelector('.getStyle');
const element2 = document.querySelector('.getStyle1');
const el1 = getComputedStyle(element1);
element2.innerHTML = `font-size: ` + el1.fontSize + ' ' + `color: ` + el1.color;
}

Атрибуты.

С помощью нестандартных атрибутов можно привязать к элементу данные, доступные в JavaScript. Это делается с помощью атрибутов с названиями, которые начинаются на data-

.hasAttribute

Проверяет наличие атрибута.
element.hasAttribute(name)

.getAttribute

Получает значение атрибута.
element.getAttribute(name)

.setAttribute

Устанавливает атрибут.
element.setAttribute(имя, значение)

.removeAttribute

Удаляет атрибут.
element.removeAttribute(name)

Пример

Есть тег < p id="attribute-text" >
Проверим есть ли в теги id, выведим id и добавим class="text"

Начальный текст.

function changeAttribute() {
const element = document.getElementById('attribute-text');
element.setAttribute('class', 'text');
element.innerHTML =
`hasAttribute('id'): ` +
element.hasAttribute('id') + ' ' +
`getAttribute('id'): ` +
element.getAttribute('id') + ' ' +
`hasAttribute('class'): ` +
element.hasAttribute('class');
}

Классы.

.className

Отвечает за значение атрибута class элемента. element.className

.classList

Возвращает псевдомассив DOMTokenList, содержащий все классы элемента.
element.classList

.classList.contains

Возвращает true/false, в зависимости от того, есть ли у элемента класс class.
element.classList.contains("class")

.classList.add

Добавляет класс.
element.classList.add("class")

.classList.remove

Удаляет класс.
element.classList.remove("class")

.classList.toggle

Если класса нет, добавляет его, если есть – удаляет.
element.classList.toggle("class")

document.createElement(tag)

Создаёт элемент с заданным тегом.

document.createTextNode(value)

Добавление и удаление элементов.

.append()

Добавляет узлы или строки в конец выбранного тега.

.prepend()

Вставляет узлы или строки в начало выбранного тега.

.before()

Вставляет узлы или строки до выбранного тега.

.after()

Вставляет узлы или строки после выбранного тега.

.replaceWith()

Заменяет выбранный тег заданными узлами или строками.

Пример
  1. 0
  2. 1
  3. 2

function addElemets() {
list_ol.before('before'); // вставить строку "before" перед < ol >
list_ol.after('after'); // вставить строку "after" после < ol >
let liFirst = document.createElement('li');
liFirst.innerHTML = 'prepend';
list_ol.prepend(liFirst); // вставить liFirst в начало < ol >
let liLast = document.createElement('li');
liLast.innerHTML = 'append';
list_ol.append(liLast); // вставить liLast в конец < ol >
}

Клонирование элементов.

.cloneNode(deep)

Клонирует элемент, если deep==true, то со всеми дочерними элементами.
element.cloneNode(deep)

.cloneNode(true)

Создаёт «глубокий» клон элемента – со всеми атрибутами и дочерними элементами.
element.cloneNode(true)

.cloneNode(false)

Клон будет без дочерних элементов.
element.cloneNode(false)

Это функция, которая первым параметром принимает название евента, а вторым параметром функцию, которая выполнится, когда мы кликнем.
element.addEventListener()

.removeEventListener()

Используется для удаления обработчика.
element.removeEventListener()

.dispatchEvent()

Генерирует событие на объекте.
element.dispatchEvent()

Делегирование событий.

event.target

Является ссылкой на объект, который был инициатором события.

event.stopPropagation()

Для остановки всплывания нужно вызвать метод.

Пример всплытия
section
form
div

Если в примере выше кликнуть на div, то последовательно выведутся alert: div → form → section. Этот процесс называется всплытием, потому что события «всплывают» от внутреннего элемента вверх через родителей, подобно тому, как всплывает пузырёк воздуха в воде.

Пример event.target

event.target – это исходный элемент, на котором произошло событие, в процессе всплытия он неизменен.
this – это текущий элемент, до которого дошло всплытие, на нём сейчас выполняется обработчик.

section
form
div

const form = document.querySelector('.second_form');
form.onclick = function (event) {
alert('target = ' + event.target.tagName + ', this=' + this.tagName);
};


Браузерные события.

События мыши.

auxclick

Кнопка указывающего устройства (любая неосновная кнопка) была нажата и отпущена на элементе.

click

Происходит, когда кликнули на элемент левой кнопкой мыши.

dblclick

Происходит, когда дважды кликнули на элемент левой кнопкой мыши.

contextmenu

Происходит, когда кликнули на элемент правой кнопкой мыши.

mouseover

Когда мышь наводится на элемент.

mouseout

Когда мышь покидает элемент.

mousedown

Когда нажали кнопку мыши на элементе.

mouseup

Когда отжали кнопку мыши на элементе.

mousemove

При движении мыши.

mouseenter

Указывающее устройство перемещено на элемент, к которому подключён обработчик.

mouseleave

Указывающее устройство перемещается от элемента, к которому подключён обработчик.

pointerlockchange

Указатель был заблокирован или отпущен.

pointerlockerror

Указатель был заблокирован по техническим причинам или из-за отказа в разрешении.

select

Выбирается какой-то текст.

wheel

Колесо указывающего устройства вращается в любом направлении.

События на элементах управления.

submit

Пользователь отправил форму < form >.

reset

Кнопка сброса нажата.

События фокуса.

focus

Пользователь фокусируется на элементе, например нажимает на < input >.

blur

Элемент потерял фокус.

Клавиатурные события.

keydown

Когда пользователь нажимает клавишу.

keyup

Когда пользователь отпускает клавишу.

keypress

Любая клавиша, кроме Shift, Fn, CapsLock, находится в нажатом положении. (Происходит непрерывно.)

События документа.

DOMContentLoaded

Когда HTML загружен и обработан, DOM документа полностью построен и доступен.

CSS events.

animationstart

CSS-анимация началась.

animationcancel

CSS-анимация прервалась.

animationend

CSS-анимация завершена.

animationiteration

CSS-анимация повторяется.

transitionstart

transitioncancel

CSS переход отменён.

transitionend

CSS переход завершён.

transitionrun

pageshow

popstate

Выполняется переход к записи журнала сессии (в некоторых случаях).

События Text Composition

compositionstart

Подготавливается композиция отрывка текста (аналогично нажатию клавиш для ввода с клавиатуры, но работает с другими входами, такими как распознавание речи).

compositionupdate

К отрывку составляемого текста добавляется символ.

compositionend

Составление отрывка текста завершено или отменено.

События печати

beforeprint

Диалоговое окно печати открыто.

afterprint

Диалоговое окно печати закрыто.

События ресурса.

Не удалось загрузить ресурс.

abort

Загрузка ресурса была прервана.

load

Ресурс и его зависимые ресурсы завершили загрузку.

loadend

Выполнение остановлено (после отправки "error", "abort" или "load").

loadstart

Выполнение началось.

progress

В ходе выполнения.

timeout

Выполнение прекращено по истечении установленного времени.

beforeunload

Окно, документ и его ресурсы будут выгружены.

unload

Выгружается документ или зависимый ресурс.

Сетевые события.

online

Браузер получил доступ к сети.

offline

open

message

error

close

resize

Размер просмотра документа изменён.

scroll

Документ или элемент были прокручены.

События буфера обмена.

cut

Выделение было вырезано и скопировано в буфер обмена.

copy

Выделение было скопировано в буфер обмена.

paste

Элемент из буфера обмена был вставлен

События Drag & Drop

drag

Элемент или выделение текста перетаскивается. (Срабатывает непрерывно каждые 350 мс).

dragend

Операция перетаскивания завершается (отпуская кнопку мыши или нажимая клавишу выхода).

dragenter

Перетаскиваемый элемент или выделенный текст попадает в допустимую цель перетаскивания.

dragstart

Пользователь начинает перетаскивать элемент или выделенный текст.

dragleave

Перетаскиваемый элемент или выделенный текст оставляет допустимую цель перетаскивания.

dragover

Выбранный элемент или текст перетаскивается над допустимой целью перетаскивания. (Срабатывает непрерывно каждые 350 мс).

drop

Элемент перетаскивается на допустимую цель перетаскивания.
